============================================
第七版修复日志
============================================

## 修复目标

修复3个老问题：
1. 中文含义输入阶段从录音改为手动输入+手写识别
2. 拼写阶段不应显示英文单词（听写不是照抄）
3. 首页"复习"按钮显示"复习词库为空"，但实际有单词

## 修复内容

### 问题1：录音无法识别 → 改为手动输入+手写

**修改文件**：`src/pages/dictation/index.tsx`

**改动**：
1. 新增状态 `meaningInput`（含义输入框内容）
2. 删除 `isRecording` 状态和 `Mic` 图标导入
3. 删除录音相关代码（toggleRecording、handleVoiceResult 函数）
4. 新增 `handleMeaningSubmit` 函数（提交中文含义）
5. 含义阶段 UI 从录音按钮替换为 Input 输入框 + 手写按钮 + 提交/不知道按钮
6. 手写识别结果根据当前 phase 填入对应输入框（spelling→spellingInput，meaning→meaningInput）

### 问题2：拼写阶段显示单词

**修改文件**：`src/pages/dictation/index.tsx`

**改动**：
- 拼写/含义阶段不再显示 `{currentWord?.word}`
- 改为显示"第 X 个单词"（灰色提示文字）

### 问题3：首页复习词库为空

**修改文件**：`src/pages/index/index.tsx`

**根因**：`loadVocabularyStats` 直接用 `Taro.getStorageSync('key') || []` 取 `.length`，但存储的是 JSON 字符串，不是数组。

**改动**：
- 兼容字符串和数组两种格式读取存储
- 增加 `Array.isArray` 检查防止取 `.length` 报错

- `pnpm build:web` ✅ 成功
- `pnpm build:server` ✅ 成功
