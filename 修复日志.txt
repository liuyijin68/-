============================================
第八版修复日志
============================================

## 修复目标

修复4个老问题：
1. 发音获取失败（多次修复无效）
2. 首页"复习"词库数量不更新
3. 听写朗读时显示英文单词
4. 含义阶段提示词"请说出中文含义"未改

## 修复内容

### 问题1：发音获取失败

**根因分析**：
- Network.ts 使用 `PROJECT_DOMAIN` 全局变量拼接 URL
- 如果 `PROJECT_DOMAIN` 指向了 localhost 或 127.0.0.1，小程序环境无法访问
- 之前的修复缺少详细错误日志，无法定位是"请求失败"还是"播放失败"

**修改文件**：`src/pages/dictation/index.tsx`

**改动**：
- playWord 函数增加详细日志：
  - 请求前打印 `[playWord] 请求发音: {word}`
  - 请求后打印 `[playWord] 响应: {完整响应JSON}`
  - 打印美式和英式音频 URL
  - 区分错误类型：`request:fail`（后端地址问题）vs 其他错误
  - 显示具体错误信息给用户

### 问题2：首页复习词库数量不更新

**修改文件**：`src/pages/index/index.tsx`

**改动**：
- 新增 `loadFromStorage` 辅助函数，统一处理 JSON 字符串和数组两种格式
- `loadVocabularyStats` 使用该辅助函数读取存储

### 问题3：听写朗读时显示英文单词

**修改文件**：`src/pages/dictation/index.tsx`

**改动**：
- playing 阶段删除 `{currentWord?.word}` 显示
- 只保留"正在朗读单词，请仔细听..."提示

### 问题4：含义阶段提示词

**修改文件**：`src/pages/dictation/index.tsx`

**状态**：已在第七版修改为"请输入中文含义："，无需额外修改。

## 关于发音问题的进一步排查

如果修复后仍无声音，请在微信开发者工具控制台查看以下日志：
1. `[playWord] 请求发音:` — 确认请求已发出
2. `[playWord] 响应:` — 确认后端返回了音频 URL
3. `[playWord] 美式URL:` — 确认 URL 格式正确
4. `[playAudioUrl] 开始播放:` — 确认前端开始播放
5. `[playAudioUrl] 播放错误:` — 确认具体播放错误

如果第1步就报错 `request:fail`，说明 PROJECT_DOMAIN 配置有问题。
如果第4步正常但无声，检查微信小程序后台的合法域名配置。
