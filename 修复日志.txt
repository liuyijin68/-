## 本次修复目标

1. **图片识别把整行当单词**：LLM 返回 JSON 数组时，每个字符串包含音标、词性、中文等脏数据
2. 确保无论 LLM 返回什么格式，都只提取纯英文单词/短语

---

## 根因分析

第三版代码的清洗逻辑只在"LLM 返回非 JSON 时（fallback）"才执行。
当 LLM 正常返回 JSON 数组时，直接使用了数组中的原始字符串（可能包含音标、中文等）。

例如 LLM 返回：
```json
["important /ɪmˈpɔːtnt/ adj. 重要的", "notice /ˈnəʊtɪs/ n. 通知"]
```

第三版代码直接用这两个字符串作为单词，导致显示错误。

---

## 修复内容

### 1. 添加统一清洗函数

文件：`server/src/dictation/dictation.controller.ts`（第36-49行）

```typescript
// Fix 第四版: 统一清洗函数 — 只取英文部分
private cleanWord(raw: string): string {
  // 1. 找到第一个 / 或中文字符的位置
  const slashIdx = raw.indexOf('/');
  const chineseIdx = raw.search(/[\u4e00-\u9fa5]/);
  let endIdx = raw.length;
  if (slashIdx !== -1 && slashIdx < endIdx) endIdx = slashIdx;
  if (chineseIdx !== -1 && chineseIdx < endIdx) endIdx = chineseIdx;
  
  // 2. 截取并清理多余字符（保留字母、空格、括号、连字符）
  let cleaned = raw.substring(0, endIdx).trim();
  cleaned = cleaned.replace(/[^a-zA-Z\s\(\)\-]/g, '').trim();
  return cleaned;
}
```

### 3. 修改 translateWord 方法

文件：`server/src/dictation/dictation.controller.ts`（第176-196行）

```typescript
// Fix 第四版: 防止空含义导致异常
private async translateWord(word: string): Promise<string[]> {
  try {
    const response = await this.llmClient.invoke(
      [{ role: 'user', content: `Translate "${word}" into Chinese. List ALL meanings as JSON array. If no translation, return ["${word}"]` }],
      { model: 'doubao-seed-2-0-lite-260215', temperature: 0.3 },
    );
    const content: string = response?.content || '';
    const jsonMatch = content.match(/\[[\s\S]*\]/);
    if (jsonMatch) {
      try {
        const arr = JSON.parse(jsonMatch[0]);
        return Array.isArray(arr) && arr.length ? arr : [word];
      } catch { return [word]; }
    }
    return [word];
  } catch { return [word]; }
}
```
