============================================
六、Bug 模拟测试（第二版）
============================================

### Bug 1：图片识别无单词 ✅

**修复方案**：
- 后端：增强 LLM 内容解析（JSON → 按行 → 按字符）
- 后端：返回 raw 字段供前端调试
- 前端：检查 code 非 200 时显示后端错误消息
- 前端：未识别到时显示 LLM raw content

**模拟路径**：
```
用户点击"识别单词" → 后端调用 LLM → LLM 返回任意格式
→ 后端尝试 JSON.parse → 失败则按行拆分 → 再失败则按字符匹配
→ 返回 words 或 raw → 前端显示结果或错误详情
```

**断言**：
- 多层 fallback 解析确保不会因格式问题返回空 ✅
- 前端能看到详细错误信息 ✅


### Bug 2：词库数据解析失败 ✅

**修复方案**：
- 统一存储格式辅助函数 `loadFromStorage` / `saveToStorage`
- 兼容字符串和数组两种格式

**模拟路径**：
```
词库管理页面存入 JSON 字符串 → 听写页面读取
→ loadWordsFromStorage 判断类型 → 字符串则 JSON.parse → 数组则直接使用
→ 成功获取单词列表 → 正常进入听写流程
```

**断言**：
- `typeof raw === 'string' ? JSON.parse(raw) : raw` 兼容两种格式 ✅
- 空词库友好提示 ✅
