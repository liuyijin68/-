# 单词听写小程序设计指南

## 产品定位

面向学生群体的英语单词听写辅助工具，通过拍照识别 + 语音交互，让听写练习更高效有趣。

## 设计风格

**气质与意象**：清晨教室里的英语课，干净的书桌，简洁的单词卡片，温暖的阳光透过窗户洒在笔记本上。界面简洁清爽，像一本精心整理的单词本。

## 配色方案

| 用途 | Tailwind 类名 | 说明 |
|------|--------------|------|
| 主色 | `bg-blue-500` / `text-blue-600` | 英语学习常用蓝色系，清新专注 |
| 辅色 | `bg-emerald-500` | 正确提示，绿色代表通过 |
| 错误色 | `bg-red-500` | 错误提示，红色醒目 |
| 背景 | `bg-gray-50` / `bg-white` | 浅灰底色，白色卡片 |
| 文字主色 | `text-gray-800` | 正文深灰 |
| 文字辅色 | `text-gray-500` | 说明文字浅灰 |

## 组件选型原则

- 通用 UI 组件优先使用 `@/components/ui/*`
- 按钮：Button
- 输入框：Input
- 卡片：Card
- 弹窗：Dialog
- 提示：Toast
- 进度：Progress

## 页面结构

### 首页 (pages/index/index.tsx)
- 上传照片区域
- Canvas 圈选界面
- 确认提交按钮
- 已选单词预览

### 听写页面 (pages/dictation/index.tsx)
- 当前单词显示（英语 + 音标）
- 进度条
- 开始听写按钮
- 录音/输入答案
- 结果反馈
- 统计页面

## 交互流程

1. 用户拍照上传 → 2. Canvas圈选范围 → 3. 确认提交 → 4. LLM识别单词 → 5. 进入听写 → 6. TTS念单词 → 7. 用户回答 → 8. 结果判断 → 9. 下一个单词

## 设计禁忌

- 禁止复杂的渐变背景
- 禁止过多的装饰元素
- 禁止花哨的动画效果
- 保持界面简洁，专注于听写功能
